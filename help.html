Language Guide

Hello, World!

For starters, here is a Hello World program. 

print "Hello, World!"

Program Structure

A LatteScript program is simply a list of subroutine definitions, including a definition for the procedure main. So what constitutes a subroutine definition? A subroutine definition is either a function definition or a procedure definition. A procedure definition looks like this: 

procedure name arg1, arg2, arg3
    print "hello procedure"

A procedure definition has the following parts, in order:

  1. procedure
  2. the procedure name
  3. (optional) a comma separated list of arguments.
  4. a newline
  5. an indented code block.

A function definition looks similar: 

function name(arg1, arg2, arg3)
    print "hello function"
    return 123

Nullary functions still require parentheses. For example: 

function f()
    return g()

function g()
    return 4

Procedures never use parentheses. For example: 

procedure p x
    q "foo", x

procedure q a, b
    print a ~ " " ~ b
    speak

procedure speak
    print "hello"

A complete LatteScript program is just a list of procedures and functions, as seen above, and additionl code statements to execute.

Variables

Variables defined outside subroutines are global, and can be assigned from within subroutines. Variables defind inside subroutines are local to their subroutines. Parameters cannot be assigned.

Identifiers

Variables, functions, and procedures all have separate namespaces, so one single identifier can have three different meanings depending on how it is used. All identifiers are case sensitive. Identifiers start with underscore or an alphabetic character, and are followed by as many alphanumeric characters or underscores as you like.

Comments

Any line beginning with zero or more spaces and a semicolon (;) is ignored.

Blocks

Blocks are consistently-indented consecutive code statements. Any amount of spacing is supported, so long as the indentation is greater than its parent indentation. The only legal indentation character is space. Literal tab characters are always a syntax error.

Decision Constructs

Conditional code is implemented using the familiar if construct. Example: 

if x < 0
    print "negative"
else if x < 10
    print "single digit"
else if x < 100
    print "double digit"
else
    print "too big"

The only mandatory part of an if construct is the if part. All else if and else branches are optional.

Looping Constructs

There are four looping constructs. First, there are the while and until loops. These loops test their conditions before executing. while executes its body while the expression is true, and until executes its expression until the expression is true. For example: 

n := 1
while n < 10
    print "Zzz... " ~ n ~ " sheep counted so far."
    n := n + 1

n := 10
until n = 0
    print n
    n := n - 1

print "Blast-off!"

The other two looping constructs are the two for loops, referred to as for each and for range. These loops both take a variable which assumes multiple values over the course of the loop. The following is an example of printing the numbers 1, 2, 3 in order using both types of for loops: 

for n from 1 to 3
    print n

for n in [1, 2, 3]
    print n

The for each version of the loop also works on text objects, giving back one character at a time. The for range loop can also use a by clause to modify the step size, such as in the following example, which prints 10, 20, and 30: 

for n from 10 to 30 by 10
    print n

Assignment

Variables can be assigned using the symbol :=

You can also assign to list access, like items@n := x

Lists

Lists are dynamic arrays, like JavaScript's Array and Java's ArrayList. Lists are created using brackets and commas, such as [1, 2, 3], [], or [[1, 2], [3, 4]]. Lists are indexed starting at one and using the @ symbol. The prefix operator # returns the length of a list. The built-in procedure append can be used to append a value to a list. Example: 

nums := []
for n from 7 to 9
    append nums, n
; nums = [7, 8, 9]

; prints 7, 8, 9
for n from 1 to #nums
    print nums@n

for n from 1 to #nums
    nums@n := "foo"
; nums = ["foo", "foo", "foo"]

; prints foo foo foo
for n in nums
    print n

Expressions

The following operators are included: + - * / ^ @ ~ < > <= >= div mod and or.
